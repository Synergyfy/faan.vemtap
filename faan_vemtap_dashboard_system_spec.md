# FAAN VEMTAP ENTERPRISE SYSTEM – FULL USER FLOW (STEP BY STEP UI GUIDE)

---

# 1. ENTRY POINT

User visits:

faan.vemtap.com

---

# 2. LOGIN PAGE (FAAN BRANDED)

This is the first screen.

Design:

• FAAN Logo at the top center  
• Title: “FAAN Passenger Engagement System”  
• Subtitle: “Secure Access Portal”  

Form Fields:
• Email Address  
• Password  

Button:
• Login

Footer:
• “Powered by VEMTAP” (small text)

No extra links. No signup. No distractions.

---

# 3. AFTER LOGIN → DASHBOARD (HOME)

This is the main overview screen.

---

## TOP SECTION (KPI CARDS)

Display large cards:

• Total Interactions Today  
• Total Feedback Received  
• Open Issues  
• Resolved Issues  
• Average Satisfaction Score (1–5)  
• Average Response Time

---

## MIDDLE SECTION (CHARTS)

• Line Chart: Feedback Trend (Daily submissions)  
• Pie Chart: Issue Categories (Cleanliness, Staff, Security, Facilities)

---

## BOTTOM SECTION (LIVE ACTIVITY)

Recent Activity Feed:

• “Complaint submitted – Terminal 2 Restroom”  
• “Feedback received – Departure Gate A”  
• “Issue resolved – Baggage Area”

---

# 4. LEFT SIDEBAR MENU

Menu Items:

• Dashboard

---

# 5. LOCATIONS PAGE

Purpose: View all airports, terminals, and zones.

---

## LEFT PANEL (TREE VIEW)

• Airport (e.g. Abuja, Lagos)  
   → Terminal  
      → Zones (Gate, Lounge, Restroom, Security Area)

---

## MAIN VIEW

When a location is selected:

Show:

• Total interactions  
• Satisfaction score  
• Open issues  
• Top complaint type

---

## HEATMAP (OPTIONAL BUT IMPORTANT)

Color system:

• Red = High complaints  
• Yellow = Medium  
• Green = Good

---

# 6. TOUCHPOINTS PAGE

Purpose: Manage all QR/NFC points.

---

## TABLE VIEW

Columns:

• Name  
• Location  
• Type (Feedback / Complaint / Incident)  
• Status (Active / Inactive)  
• Total Interactions

---

## ACTION BUTTONS

• View QR Code  
• Download QR  
• Edit  
• Disable

---

## CREATE NEW TOUCHPOINT

Fields:

• Title  
• Location  
• Department  
• Type  
• Form Template

System generates:

• Unique Link  
• QR Code  
• NFC Link

---

# 7. SUBMISSIONS PAGE

Purpose: View all passenger inputs.

---

## TABLE VIEW

Columns:

• Submission ID  
• Type  
• Location

• Status  
• Department

---

• Add internal note

# 8. ISSUE MANAGEMENT PAGE

Purpose: Track issue progress.

---

## KANBAN BOARD

• Pending
