# Frontend Integration Guide: Automated Reporting System

This guide outlines how to integrate the backend Automated Reporting System into the FAAN VEMTAP dashboard.

## 1. API Base URL
All endpoints are relative to: `${API_URL}/reports/automated`

---

## 2. Enums and Types

You should define these enums and interfaces in your frontend `types/api.ts` or similar file.

```typescript
export enum SystemReportType {
  DAILY = 'DAILY',
  WEEKLY = 'WEEKLY',
  MONTHLY = 'MONTHLY',
}

export enum SystemReportCategory {
  GENERAL = 'GENERAL',
  INTERNAL = 'INTERNAL',
}

export interface DistributionItem {
  name: string;
  count: number;
  percentage: number;
}

export interface ChartData {
  title: string;
  data: DistributionItem[];
  explanation: string;
}

export interface SystemReport {
  id: string;
  reportType: SystemReportType;
  category: SystemReportCategory;
  date: string; // ISO Date
  weekRange?: string | null;
  month?: string | null;
  content: string; // AI Generated Summary
  metrics: any; // Raw statistical data
  graphData: ChartData[];
  pdfFileUrl?: string | null;
  locationId?: string | null;
  departmentId?: string | null;
  createdAt: string;
}
```

## 3. Endpoints

### 3.1 Generate Report (On-Demand)
Triggers the AI and data aggregation engine to create a new report.

- **URL**: `POST /generate`
- **Roles**: `SUPER_ADMIN`, `LOCATION_ADMIN`
- **Payload**:
```typescript
interface GenerateReportPayload {
  reportType: SystemReportType;
  category: SystemReportCategory;
  locationId?: string;
  departmentId?: string;
  date?: string; // Optional: specific date to report on
}
```

### 3.2 List Reports
Retrieves a list of previously generated automated reports.

- **URL**: `GET /`
- **Query Parameters**:
```typescript
interface QueryReports {
  reportType?: SystemReportType;
  category?: SystemReportCategory;
  locationId?: string;
  departmentId?: string;
}
```

### 3.3 Get Report Details
Fetches the full content and graph data for a specific report.

- **URL**: `GET /:id`

### 3.4 Download PDF
Returns a professional, branded PDF file.

- **URL**: `GET /:id/pdf`
- **Response**: `application/pdf` binary stream.
- **Tip**: Use `window.open(url)` or an anchor tag with `download` attribute.

### 3.5 Share Report via Email
Sends the report as a PDF attachment to a specified email address.

- **URL**: `POST /:id/share`
- **Payload**:
```typescript
{
  "email": "recipient@example.com"
}
```

## 4. Example Integration (React + Axios)

### Fetching Reports
```tsx
const [reports, setReports] = useState<SystemReport[]>([]);

useEffect(() => {
  const fetchReports = async () => {
    const response = await axios.get('/reports/automated');
    setReports(response.data);
  };
  fetchReports();
}, []);
```

### Triggering Manual Generation
```tsx
const handleGenerate = async () => {
  setLoading(true);
  try {
    await axios.post('/reports/automated/generate', {
      reportType: SystemReportType.DAILY,
      category: SystemReportCategory.INTERNAL
    });
    // Refresh list...
  } finally {
    setLoading(false);
  }
};
```

### Downloading PDF
```tsx
const downloadPdf = (id: string) => {
  const url = `${process.env.NEXT_PUBLIC_API_URL}/reports/automated/${id}/pdf`;
  window.open(url, '_blank');
};
```

## 5. Notes for UI/UX
1. **AI Summary**: The `content` field contains Markdown-friendly text generated by Gemini. Use a Markdown component to render it beautifully.
2. **Graphs**: The `graphData` provides pre-calculated percentages. You can use these to feed any charting library (Recharts, Chart.js) directly.
3. **Empty States**: If `totalSubmissions` is 0, the AI summary will reflect this. Ensure the UI handles the "No Data" scenario gracefully.
